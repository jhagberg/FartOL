<objective>
Build the transport + station layers: a Node `serialport@13`-based byte-stream transport that replaces upstream's WebUSB/libusb driver, plus a simplified multiplexer/send-queue/state-machine that orchestrates the BSM7/8 handshake (SET_MS -> GET_SYS_VAL -> SET_SYS_VAL diff), prepends WAKEUP to every command (codex review #11), wires the parser's structured `onFrameError` callback directly (codex review #1 — no console.warn intercept), and dispatches card-insert frames to BaseSiCard.detectFromMessage. Land everything behind interfaces that accept a FakeSerialTransport so the unit tests run zero-hardware in CI.

**KEY DESIGN CHOICES FROM CODEX REVIEW:**
- **Review #1 (HIGH):** Plan 02 emits structured FrameError via the parseAll callback. This plan wires the callback in SiTargetMultiplexer — no console.warn interception. SiMainStation re-emits as `'frameError'` event.
- **Review #2 (HIGH):** Station-level tests cover SI9, SI10, AND SIAC insertion paths (not just SI5). Modern-card tests assert the GET_SI8 page-4 request sequence and the SI8_DET dispatch.
- **Review #5 (HIGH):** Shared transport errors live in `packages/sportident/src/transport/errors.ts`, created as Task 0 (Wave 0 of this plan) BEFORE SerialTransport or SiSendTask reference them. This fixes the typecheck ordering issue where Task 1 used to reference DeviceClosedError before Task 2 created it.
- **Review #11 (MEDIUM):** `SiTargetMultiplexer.sendMessage()` prepends WAKEUP to every command, not just during the initial handshake. The multiplexer's `_renderForWire(message)` helper returns `[proto.WAKEUP, ...render(message)]`.

Purpose: This is the "transport surgery" RESEARCH §"Reality check on the WebSerial → serialport framing" called out as bigger than CONTEXT implied. We REPLACE all of `sportident-{webusb,node-usb}/src/*` with one ~150 LOC `SerialTransport.ts` plus a heavily-simplified `SiTargetMultiplexer` (Direct-only, no SET_MS dance on every command). Phase 0 needs exactly one reader on `/dev/ttyUSB0`, one card at a time — no multi-target complexity.

Output: A working transport + station stack. Given a FakeSerialTransport that returns pre-canned response frames for SET_MS / GET_SYS_VAL / SET_SYS_VAL / SI5_DET / GET_SI5 / SI8_DET / GET_SI8, the SiMainStation completes the handshake, detects SI5/SI9/SI10/SIAC insertions, reads the cards (modern cards exercising page-4 read), and emits typed events. Real-hardware bring-up is Plan 06's smoke script.
</objective>
