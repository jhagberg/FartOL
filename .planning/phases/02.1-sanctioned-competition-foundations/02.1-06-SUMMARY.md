# Phase 2.1 Plan 06: Kvar-i-skogen — Check-unit Backup Readout + Diff View

**One-liner:** BSF8 backup memory readout (GET_SYS_VAL + GET_BACKUP protocol) with started-minus-returned diff view and safety-call summary copyable to clipboard.

## What Was Built

### Task 1: readBackupMemory — BSF8 backup readout protocol (TDD)

**packages/sportident/src/SiStation/readBackup.ts**

- `parseBackupBlock(block, recLen)` — parses a 128-byte GET_BACKUP response block into `BackupRecord[]` at REC_LEN=8 stride. Card number at BC_CN offset (big-endian uint32); skips zero-card empty slots.
- `parseMemPointer(sysValParams)` — extracts big-endian uint32 memory pointer from offset 0x1C..0x1F of the GET_SYS_VAL response.
- `parseOverflowFlag(sysValParams)` — reads bit 0 of byte 0x1B for the hardware overflow flag.
- `readBackupMemory(station)` — orchestrates GET_SYS_VAL → loop GET_BACKUP. Hard cap: MAX_ITERATIONS=512 (T-02.1-11 DoS mitigation; BSF8 max 32KB = 256 blocks, 2x safety margin). Returns `{ records, overflow }`.

7 TDD tests cover all pure parsing functions + mock station integration + loop-cap behavior. All synthetic — no hardware required.

### Task 2: Checkunit snapshot route + KvarISkovenView

**apps/edge/src/routes/checkunit.ts**

- `POST /api/competitions/:id/checkunit/snapshot[?reader=<position>]`
- Returns `{ cardNumbers, returnedCardNumbers, overflow, readCount }`
- `returnedCardNumbers`: physically returned cards (card_read event with non-null finish field). Computed OK/MP/DNF status intentionally excluded — a runner can be marked DNF manually without being physically back.
- Reader selection: `?reader=left|right` matches lifecycle by position label; defaults to first connected reader.
- Error responses: 404 (no competition), 503 (no reader / station null), 500 (serial error)

**apps/edge/src/bin/fartola.ts** — Added `BridgeLifecycle.getStation(): SiMainStation | null`

**apps/edge/src/routes/sessions.ts** — Extended `bridgeLifecycles` type declaration with `getStation()` method

**apps/web/src/lib/screens/KvarISkovenView.svelte**

- "Hämta stämpeldata" button triggers POST checkunit/snapshot
- Diff: started (check-unit cards) MINUS returned (physical finish reads) MINUS confirmed-safe (manual override)
- Missing table: name, club, class, start time, elapsed since start (midnight-safe modulo)
- Unknown cards section for SI numbers not in the competitor table
- "Bekräftad säker" button lets operator mark a runner safe without a finish read
- Safety-call summary: count badge + name list string
- Copy-to-clipboard button for the summary text
- Overflow warning when backup memory wrapped around

**Navigation:** "Kvar i skogen" nav item added to Sidebar; alert-triangle icon added to Icon.svelte; +layout.svelte routing updated.

**i18n:** 24 keys added to sv.json and en.json (kvariskov.* namespace + nav.kvariskov).

### Task 3: Hardware checkpoint (non-blocking)

Manual verification against a real BSF8 check unit is required but cannot be performed automatically. Gated as `non-blocking` per plan frontmatter. The operator should:

1. Plug a BSF8 check unit (mode 0x02) into the bridge laptop
2. Stamp a few known cards through the check unit
3. Navigate to /competition/:id/kvar-i-skogen
4. Click "Hämta stämpeldata"
5. Verify the card numbers match the stamped cards
6. Verify the diff shows the correct "still in forest" list
