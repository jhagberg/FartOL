# Phase 02.1 Plan 03: IOF XML 3.0 StartList Export + Import Summary

**One-liner:** IOF XML 3.0 StartList export + import with two-tier SI-card/name matching, XSD validation, and thermal class-list printing.

## What Was Built

### Task 1 (TDD) — buildStartListXml + validateAndBuildStartList

Added to `apps/edge/src/xml/iofExport.ts`:

- `buildStartListXml(input: StartListInput): StartListBuildResult` — builds a valid IOF XML 3.0 `<StartList>` document from drawn competitor data. Excludes competitors without a `startTimeMs`. Emits UTC ISO timestamps with Z suffix. Wraps club in `<Organisation type="Club">`. Omits `@status` (StartList has no such attribute in the XSD). Competitors with status CANCEL are excluded entirely (PersonRaceStart has no Status element).
- `validateAndBuildStartList(input): Promise<ValidatedStartListBuildResult>` — runs the XML through the existing xmllint-wasm XSD pipeline (SC#6 contract).
- Public types: `StartListCompetitor`, `StartListClass`, `StartListInput`, `StartListSummary`, `StartListBuildResult`, `ValidatedStartListBuildResult`.

TDD commits: RED `921d9b4`, GREEN `323cce3` (D-14 regression test `6b` added in GREEN to lock `buildResultListXml` CANCEL→Cancelled path).

### Task 2 — importStartList, routes, thermal print

**`apps/edge/src/xml/iofImport.ts`** (new):
- `importStartList(xml: string): ImportedStartEntry[]` — parses IOF XML 3.0 StartList. Excludes entries without parseable StartTime. Extracts SI card from `ControlCard[@punchingSystem='SI']` and Eventor person ID from `Person.Id[@type='Eventor']`. `processEntities:false` blocks DOCTYPE/ENTITY injection.

**`apps/edge/src/routes/import.ts`** (extended):
- `POST /api/competitions/:id/import/startlist` — multipart XML upload. Two-tier matching: SI card exact match → auto-write in transaction; name+class fuzzy match → return `fuzzyMatches[]` for operator confirmation. Duplicate name detection per class prevents silent ambiguous writes. Returns `{ exact, fuzzy, unmatched, fuzzyMatches }`.
- `POST /api/competitions/:id/import/startlist/confirm` — accepts `{ matches: [{ competitorId, startTimeMs }] }`. Idempotent: skips entries where `existing.startTimeMs === m.startTimeMs`. Returns `{ applied, alreadyApplied }`.

**`apps/edge/src/routes/export.ts`** (extended):
- `GET /api/competitions/:id/export/startlist` — loads competitors with `startTimeMs`, groups by class, calls `validateAndBuildStartList`, streams XML with `Content-Disposition: attachment; filename="${slug}-startlist.xml"`. Returns 400 + xsd_invalid errors if validation fails (SC#6).

**`apps/edge/src/print/templates.ts`** (extended):
- `renderStartListTemplate(printer, className, date, entries)` — thermal printer renderer for class start lists. Sorts entries by `startTimeMs` ascending. Emits bib/name left-aligned, HH:MM:SS time right-aligned via `leftRight()`. Club on indented second line when present.
- `formatStartTime(epochMs)` — UTC HH:MM:SS formatter.

## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 1 - Bug] XSD sequence order — Organisation before Start**
- **Found during:** Task 1 GREEN phase — XSD validation failed on first attempt
- **Issue:** `fast-xml-parser` serialises JS object keys in insertion order. The initial `PersonStartNode` built `{ Person, Start }` then assigned `Organisation` as a property, producing `<Organisation>` after `<Start>` in XML, violating the XSD sequence constraint.
- **Fix:** Build two separate object literals: `{ Person, Organisation, Start }` (when club present) and `{ Person, Start }` (no club), selected conditionally. Insertion order matches XSD.
- **Files modified:** `apps/edge/src/xml/iofExport.ts`
- **Commit:** `323cce3`

**2. [Rule 1 - Bug] CANCEL competitors in StartList — no Status element**
- **Found during:** Task 1 GREEN phase
- **Issue:** Plan described emitting `<Status>Cancelled</Status>` for CANCEL competitors in StartList. However, the IOF XSD `PersonRaceStart` sequence does NOT include a Status element (unlike `PersonRaceResult` which does). Emitting Status would fail XSD validation.
- **Fix:** CANCEL competitors are excluded from the StartList entirely. Added test 6 (`CANCEL competitor excluded`) and regression test 6b (`buildResultListXml CANCEL still maps to Cancelled`, D-14 lock).
- **Files modified:** `apps/edge/src/xml/iofExport.ts`, `apps/edge/src/xml/iofExport.test.ts`
- **Commit:** `323cce3`

**3. [Rule 2 - Missing critical functionality] Duplicate name detection in fuzzy matching**
- **Found during:** Task 2 implementation
- **Issue:** The plan's fuzzy match design did not specify what to do when multiple competitors share the same name in the same class (e.g., two "Anna Nilsson" entries). Without detection, the import would silently pick one arbitrarily.
- **Fix:** Added duplicate detection per class. When `matchingCompetitors.length > 1`, the entry is returned in `fuzzyMatches` with `duplicates: true` flag, requiring explicit operator selection.
- **Files modified:** `apps/edge/src/routes/import.ts`
- **Commit:** `f206ca2`
