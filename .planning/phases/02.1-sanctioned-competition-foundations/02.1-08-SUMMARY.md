# Plan 02.1-08 Summary: Eventor results + startlist push

## What was built

**Task 1: pushToEventor function + routes**
- `pushToEventor()` in `apps/edge/src/eventor/push.ts` — creates PKZIP archive via yazl, POSTs with ApiKey header + Content-Type: application/zip, parses response XML for result URL
- Retry with exponential backoff (1s, 2s, 4s) on transient errors (429/500/502/503); immediate throw on 400/401/403/404
- Two POST routes in `apps/edge/src/routes/eventorPush.ts`:
  - POST /api/competitions/:id/eventor/push-results — builds ResultList XML, pushes, returns { url }
  - POST /api/competitions/:id/eventor/push-startlist — builds StartList XML, pushes, returns { url }
- Both check API key first; return 400 { error: 'no_api_key' } when missing
- 7 test cases pass (PKZIP magic bytes, ResultListUrl/StartListUrl parsing, 403 throw, timeout, retry on 502, no-retry on 403)

- Sidebar nav entry with "Eventor" badge
