## Decisions Made

- **RunnersListView instead of RegistrationView** — RegistrationView is the walk-up card-scan desk (no competitor list). Start-time display belongs on the runners roster (RunnersListView). Documented as deviation.
- **Sidebar.svelte instead of CompetitionNav.svelte** — CompetitionNav.svelte does not exist; the plan reference was for a component that was never created. Navigation is managed in Sidebar.svelte + layout.svelte routing.
- **Plain `<input>` for inline per-runner edit** — The `<Input>` component's `bind:value` rejected `string | undefined` (Record indexing). Using plain `<input value={editVal} oninput={...}>` avoids the type issue cleanly.

## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 2 - Missing Critical] Exposed start_time_ms through CompetitorDTO**
- **Found during:** Task 2 (start-time column)
- **Issue:** `start_time_ms` has been in the DB schema since Plan 01 but was never added to `CompetitorDTO` in shared-types. The SPA could never display it.
- **Fix:** Added `start_time_ms: z.number().int().nonnegative().nullable()` to CompetitorDTO. Updated competitorRowToDTO and two inline DTO constructions in competitors.ts.
- **Files modified:** packages/shared-types/src/dtos.ts, apps/edge/src/routes/competitors.ts
- **Verification:** Edge typecheck passes; web typecheck passes
- **Committed in:** de6be12 (Task 2 commit)

**2. [Rule 2 - Missing Critical] Exposed timing_format through CompetitionDTO**
- **Found during:** Task 2 (D-17 tenths display)
- **Issue:** `timing_format` column in DB from Plan 01 was not in CompetitionDTO or CompetitionPatchInput — no UI path to read or update it.
- **Fix:** Added `timing_format: z.enum(['seconds', 'tenths']).default('seconds')` to CompetitionDTO; `.optional()` variant to CompetitionPatchInput. Updated competitionRowToDTO in competitions.ts and export.ts; added PATCH handler field; fixed iofExport.test.ts fixtures.
- **Files modified:** packages/shared-types/src/dtos.ts, apps/edge/src/routes/competitions.ts, apps/edge/src/routes/export.ts, apps/edge/src/xml/iofExport.test.ts
- **Verification:** Edge typecheck passes; all iofExport tests pass
- **Committed in:** de6be12 (Task 2 commit)

**3. [Rule 3 - Blocking] LottningView inline edit used plain `<input>` instead of `<Input>` component**
- **Found during:** Task 1 (LottningView development)
- **Issue:** `bind:value={editingStartTime[runner.id]}` — TypeScript rejects `string | undefined` assignment to `bind:value` of type `string | number`. Record indexing gives `string | undefined`.
- **Fix:** Replaced `<Input>` component with plain `<input value={editVal} oninput={...}>` where `editVal = editingStartTime[runner.id] ?? ''`.
- **Files modified:** apps/web/src/lib/screens/LottningView.svelte
- **Verification:** svelte-check passes; no new type errors
- **Committed in:** 2e0e99c (Task 1 commit)

**4. [Scope adjustment] Start-time chip added to RunnersListView, not RegistrationView**

- **Issue:** Plan says "RegistrationView competitor table" but RegistrationView is a walk-up card-scan desk with no competitor list.
- **Fix:** Added amber start-time chip to RunnersListView which is the actual competitor roster.
- **Files modified:** apps/web/src/lib/screens/RunnersListView.svelte

**5. [Scope adjustment] Lottning nav link added to Sidebar.svelte, not CompetitionNav.svelte**
