## Accomplishments

- classCache.ts fetches MeOS REST `?get=class` endpoint, parses `<cls id="N">Name</cls>` XML, builds `Map<string, number>`, caches for 5 minutes

- 5s fetch timeout (T-02.1-18 DoS mitigation)

## Decisions Made

- MeOS host derived from `req.socket.remoteAddress` — the MIP polling client IS MeOS, so its source IP is the correct host. IPv6-mapped IPv4 addresses (::ffff:x.x.x.x) stripped to plain IPv4 for URL construction.
- Test helpers (`resetClassCacheForTest`, `getClassCacheForTest().seed/expireNow`) exported from classCache.ts and used in both classCache.test.ts and mip.test.ts. No test-framework coupling in production path.
- Test isolation: each test that depends on cache state calls `resetClassCacheForTest()` explicitly in `beforeEach` or at test start — avoids hidden inter-test dependencies from the shared module-level cache.
