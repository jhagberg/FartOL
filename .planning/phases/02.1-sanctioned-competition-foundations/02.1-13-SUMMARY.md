## What Was Built

### Task 1: Quality Fixes

**DQ punch contamination fix (`reduce.ts`)**
When `manual_status_set` fires with `payload.status === 'DQ'`, the reducer now zeroes `missing_codes`, `extra_codes`, `out_of_order_codes`, `latest_punches`, and `elapsed_time_ms`. Previously, stale punch data from a prior card_read leaked into the DQ display, making it appear the runner had visited controls they hadn't.

Also fixed `clear_manual_status`: when clearing a DQ override, `latest_punches` must be restored from `card_read_history` before re-running `detectStatus`, otherwise the status re-detection sees empty punches and returns MP instead of the correct status.

**LatestReadCard auto-DNF vs manual-DNF distinction**
- Added `manual_status` field to `HistoryRow` (readout.ts), `ReadoutHistoryRow` (readout-types.ts), `latestReadProp` derived object (ReadoutView.svelte), and `Read` interface (LatestReadCard.svelte)
- Rewrote `isOverridden(s)` → `isOverridden(r: Read)` checking `r.manual_status !== null` — truthfully reflects operator intent rather than guessing from status string

**StatusPill unique aria-describedby IDs (`StatusPill.svelte`)**
- Module-level `let instanceCount = 0`; per-instance `const instanceId = ++instanceCount` captured at script evaluation time
- `aria-describedby` and tooltip `id` now use `status-tip-${instanceId}` — unique per pill on page

**ClearManualStatusInput strict validation**
- `packages/shared-types/src/dtos.ts`: changed `z.object({}).passthrough()` to `z.object({}).strict()`
- `apps/edge/src/routes/manual.ts`: added `ClearManualStatusInput.safeParse(req.body ?? {})` with 400 rejection in `/clear-status` handler
- Added test in `manual.test.ts`: POST with unexpected field returns 400

### Task 2: MeOS SQL Dump Replay Harness

**`meosReplay.ts`** exports:
- `importMeosDump(dumpPath, handle, competitionId)`: parses oClass / oRunner / oCard tables from MySQL dump syntax using a character-by-character SQL scanner; inserts into fartOLa DB
- `replayCardReads(handle, competitionId)`: inserts card_read events via `insertEvent` with `REPLAY_NODE_ID = 'meos-replay-node'`
- `validateResults(handle, competitionId, expectedResults)`: runs `loadCompetitionInputs` + `reduce()`, compares status + elapsed (1 second tolerance) + missing_codes

**`meosReplay.test.ts`** covers:
- Import: oClass/oRunner/oCard rows round-trip correctly
- Replay: card_read events produce correct OK / MP / DQ statuses
- Validation: mismatches detected, matches confirmed
- Real dump test: skipped when `apps/edge/test-fixtures/meos-4klubbs-2026-05-20.sql` absent (T-02.1-28)

**`.gitignore`**: added `apps/edge/test-fixtures/meos-*.sql` to prevent PII leakage from real event dumps.

## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 1 - Bug] DQ clear reverting to MP instead of OK**
- **Found during:** Task 1 testing
- **Issue:** `clear_manual_status` handler re-ran `detectStatus` after DQ had zeroed `latest_punches`. With empty punches, all controls appeared missing → MP.
- **Fix:** Added restoration logic: if `latest_punches.length === 0` and `card_read_history` has a non-empty last entry, restore punches from history before detection.
- **Files modified:** `apps/edge/src/projection/reduce.ts`
- **Commit:** da2c2e7

**2. [Rule 1 - Bug] extractRows SQL scanner split on semicolons inside quoted strings**
- **Found during:** Task 2 testing (oCard with punch string `'31:36660;32:36720'`)
- **Issue:** Original regex `(.+?)(?:;|$)` with `gis` flags stopped at `;` inside the punch values string, producing truncated VALUES clauses and 0 parsed card rows.
- **Fix:** Replaced regex with character-by-character scanner that tracks single-quote state, only treating `;` as statement delimiter when outside a quoted string.
- **Files modified:** `apps/edge/src/test/meosReplay.ts`
- **Commit:** 1ea4247

**3. [Rule 1 - Bug] Race-phase gate blocking card reads in test (all PEND)**
- **Found during:** Task 2 testing
- **Issue:** Test competition used `raceStartedAtMs: null` (pre-race). The reducer treats all card_reads as identity scans (PEND) when race hasn't started.
- **Fix:** Set `raceStartedAtMs: 0` in test competition seed so the race is "started at epoch" and all replayed reads score.
- **Files modified:** `apps/edge/src/test/meosReplay.test.ts`
- **Commit:** 1ea4247

**4. [Rule 3 - Blocking] Static import of DB schema in non-async test callback**
- **Found during:** Task 2 testing (TransformError line 218)
- **Issue:** `await import('../db/schema.ts')` inside a non-async `test()` callback caused a TransformError.
- **Fix:** Moved to static top-level imports.
- **Files modified:** `apps/edge/src/test/meosReplay.test.ts`
- **Commit:** 1ea4247

**5. [Rule 3 - Blocking] SQLITE_CONSTRAINT_NOTNULL on competitions insert**
- **Found during:** Task 2 testing
- **Issue:** `competitions.createdAtMs` is `notNull()` with no default — missing from test seed.
- **Fix:** Added `createdAtMs: Date.now()` to all competition inserts.
- **Files modified:** `apps/edge/src/test/meosReplay.test.ts`
- **Commit:** 1ea4247
