# Phase 02.1 Plan 02: Draw algorithms + lottning route + PATCH class route

**One-liner:** Iterative SOFT club-blocking draw + Random/Simultaneous modes + transactional lottning route writing start_time_ms + PATCH class route for maxTimeSec.

## What was built

### Task 1: Draw algorithms (TDD, GREEN passed)

Three pure draw functions in `apps/edge/src/draw/`:

- **`types.ts`** — `DrawRunner { id, club }`, `DrawSlot = DrawRunner | null`, `DrawResult { order, adjacencyCount }`
- **`soft.ts`** — `drawSOFT()`: iterative round-robin club-blocking algorithm re-authored from oEventDraw.cpp logic. Groups runners by club, sorts groups by size descending, shuffles within each group, then does iterative round-robin picks. Vacant slots distributed evenly via Bresenham-style spacing. Returns `{ order, adjacencyCount }`. Uses `crypto.randomInt` (CSPRNG) by default; accepts `rngFn` for deterministic tests.
- **`random.ts`** — `drawRandom()`: Fisher-Yates shuffle. Same `rngFn` injection.
- **`simultaneous.ts`** — `drawSimultaneous()`: returns input order unchanged (start time assignment happens in the route).

**Key design decisions:**
- No recursion in SOFT (addresses T-02.1-05 DoS concern for large classes)
- `rngFn` injection enables seeded deterministic tests (GPT MEDIUM: flaky tests fix)
- Impossibility-aware: when one club > half the field, SOFT places same-club runners as far apart as possible, reports `adjacencyCount > 0`, never crashes

### Task 2: Lottning route + PATCH class route (TDD, GREEN passed)

**`apps/edge/src/routes/lottning.ts`:**
- `POST /api/competitions/:id/lottning/:classId` — draw + write start times
  - Zod validation: mode enum, firstStartMs ≥ 0, intervalSec with refinement (> 0 for SOFT/Random, any value for Simultaneous) — T-02.1-04b
  - Cross-competition pre-flight: class must belong to the competition → 404
  - Calls `drawSOFT`, `drawRandom`, or `drawSimultaneous` based on mode
  - Transactional write: wipes all start_time_ms for the class first (D-07 re-lotta), then assigns new times; also updates `classes.first_start_ms` and `classes.start_interval_sec`
  - Calls `projectionStore.markDirty(competitionId)` after commit
  - Returns `201 { drawn: N }` where N = count of non-null (non-vacant) slots
- `GET /api/competitions/:id/lottning/:classId` — fetch sorted start list

**`apps/edge/src/routes/classes.ts`** (extended with PATCH):
- `PATCH /api/competitions/:id/classes/:classId` — update `maxTimeSec` (D-08)
  - Zod: `{ maxTimeSec: number | null }` (strict schema)
  - Cross-competition pre-flight
  - Transactional UPDATE + markDirty
  - Returns `200 { ok: true }`

**`apps/edge/src/server.ts`:** Registered `registerLottningRoutes` at Phase 2.1 route block.
