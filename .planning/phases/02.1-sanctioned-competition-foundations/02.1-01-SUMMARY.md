## What Was Built

**Task 1: Schema Migration 0007**
- Migration SQL adds `first_start_ms`, `start_interval_sec`, `max_time_sec` to `classes`; `start_time_ms` to `competitors`; `liveresultat_id`, `liveresultat_pwd`, `eventor_event_id`, `timing_format` to `competitions`; and creates the `course_replacements` table with unique index on `(course_id, control_code, alternative_code)`.
- `EventPayload` union extended with `leg_voided` and `leg_unvoided` discriminated arms.
- Migration journal entry idx:7, tag `0007_phase2_1_schema`.

**Task 2: Reducer Extensions (TDD RED → GREEN)**
- `CompetitorView` gains `voided_legs: number[]` and `start_time_ms: number | null`.
- `ReduceInput` gains optional `replacementControls: ReadonlyMap<string, ReadonlyMap<number, number[]>>`.
- MAX auto-compute gate: after `detectStatus`, if class has `maxTimeSec` and `elapsed_time_ms / 1000 > maxTimeSec`, status is promoted to `MAX`. Manual overrides still win.
- Voided-leg elapsed recomputation: post-pass after the event loop recomputes elapsed by subtracting `min(actual_leg_sec, max_seconds)` for each voided control. Post-pass ensures event ordering doesn't affect projection.
- Replacement controls: single-level Map lookup rewrites the expected control code list before `detectStatus` — if the punched code is in the alternatives list, the expected code is substituted so `detectStatus` sees a match. No chaining.
- Loader extended to load `course_replacements` from DB and build the `replacementControls` map.

**Task 3: Void-Leg Routes + Idempotency**
- `POST /api/competitions/:id/competitors/:competitorId/void-leg`: Zod validates `control_code` (int), `max_seconds` (int or null), optional `reason`. Cross-competition pre-flight. Inserts `leg_voided` event. Returns 201.
- `POST /api/competitions/:id/competitors/:competitorId/unvoid-leg`: Validates `control_code`. Inserts `leg_unvoided` event. Returns 201.
- `POST /status` idempotency: reads current projection; if `view.manual_status === requested status`, returns 200 without inserting event.
- `POST /clear-status` idempotency: if `view.manual_status === null`, returns 200 without inserting event.
