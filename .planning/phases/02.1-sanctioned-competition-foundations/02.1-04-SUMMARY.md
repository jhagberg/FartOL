## What Was Built

### Task 1: Multi-serial CLI + BridgeLifecycle array

**CLI changes (`fartola.ts`):**
- New `--serial <path>[:<position>]` flag: repeatable, position label optional
- `--serial-path <path>` kept as deprecated backward-compatible alias (position=null)
- Default `/dev/ttyUSB0` when neither flag is given (unchanged behavior)
- Duplicate serial path detection: `resolve()` comparison + `process.exit(1)` with fatal error
- `CliOpts.serialPath: string` replaced by `serialPaths: SerialPathEntry[]`
- `SerialPathEntry` interface: `{ path: string; position: string | null }`
- `parseSerialEntry()` helper: splits `path:position` on the last colon

**BridgeLifecycle changes:**
- `position: string | null` as fifth constructor parameter (default null)
- `isConnected: boolean` and `lastPunchAt: number | null` private fields
- `status(): BridgeReaderStatus` method returns snapshot for health route
- `teardownCurrent()` now sets `isConnected = false`
- `connectionChanged` handler sets `isConnected = true/false` on open/close
- Broadcast wrapper intercepts `card_read` events to: (a) set `lastPunchAt`, (b) inject `reader_position` into payload when position is set

**main() changes:**
- `const lifecycles: BridgeLifecycle[] = []` replaces single `lifecycle` variable
- Loop over `opts.serialPaths` to create one lifecycle per entry
- `app.reconnectBridge` now calls `Promise.all(lifecycles.map(lc => lc.reconnectNow()))`
- `app.bridgeLifecycles = lifecycles` (empty array in --no-bridge mode)
- Shutdown loops over all lifecycles

**SQLite:**
- `busy_timeout = 5000` added to `openDatabase()` (db/index.ts) for concurrent multi-reader WAL writes

**Module augmentation (sessions.ts):**
- `FastifyInstance.bridgeLifecycles` declared as structural interface type to avoid circular imports with bin/fartola.ts

**Server (server.ts):**
- `app.decorate('bridgeLifecycles', [])` added before `registerHealthRoute()` using `hasDecorator()` guard for idempotency

**Tests (8 new cases in fartola.test.ts):**
- `--serial` with position → correct path + position
- `--serial` without position → position=null
- Multiple `--serial` flags → multiple serialPaths entries
- `--serial-path` backward compat → position=null
- Default serialPaths when no flag given
- Duplicate path → `process.exit(1)` + stderr mention

### Task 2: Health endpoint per-reader status

**`packages/shared-types/src/dtos.ts`:**
- `ReaderStatusDTO` schema: `{ path, position, connected, last_punch_at }`
- `HealthDTO` extended with `readers: z.array(ReaderStatusDTO)` field

**`apps/edge/src/routes/health.ts`:**
- Reads `app.bridgeLifecycles ?? []`
- Maps each lifecycle's `status()` snapshot to wire shape (camelCase → snake_case for `last_punch_at`)
- Returns `readers: []` when `--no-bridge` or no bridge configured

## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 2 - Missing Critical] SQLite busy_timeout not set**
- **Found during:** Task 1, reviewing db/index.ts
- **Issue:** Plan specified WAL + busy_timeout=5000 for concurrent multi-reader writes, but db/index.ts only set WAL without busy_timeout
- **Fix:** Added `sqlite.pragma('busy_timeout = 5000')` to openDatabase()
- **Files modified:** apps/edge/src/db/index.ts
- **Commit:** 68dad2a

**2. [Rule 3 - Blocking] broadcast.ts does not exist**
- **Found during:** Task 1
- **Issue:** Plan referenced `apps/edge/src/ws/broadcast.ts` as a file to modify for reader_position, but this file does not exist (broadcast happens via `app.wsBroadcast()` decoration). The broadcast wrapper pattern inside BridgeLifecycle achieves the same result without modifying a non-existent file.
- **Fix:** Implemented reader_position injection in the lifecycle's broadcast wrapper closure instead
- **Files modified:** apps/edge/src/bin/fartola.ts
- **Commit:** 68dad2a

**3. [Rule 2 - Missing Critical] app.bridgeLifecycles not initialized for no-dbHandle builds**
- **Found during:** Task 2, health route testing
- **Issue:** buildServer() without dbHandle (used in tests + walking skeleton mode) would not have the `bridgeLifecycles` decoration, causing health route to fail
- **Fix:** Added `app.decorate('bridgeLifecycles', [])` before `registerHealthRoute()` with `hasDecorator()` guard so it's always present
- **Files modified:** apps/edge/src/server.ts
- **Commit:** 68dad2a

**4. [Rule 3 - Blocking] TypeScript circular import risk**
- **Found during:** Task 1, adding bridgeLifecycles to FastifyInstance augmentation
- **Issue:** Importing `BridgeLifecycle` class type from bin/fartola.ts in sessions.ts would create circular dependencies (sessions.ts is registered by server.ts, which is separate from the bin layer)
- **Fix:** Declared bridgeLifecycles as a structural anonymous interface type in the module augmentation
- **Files modified:** apps/edge/src/routes/sessions.ts
- **Commit:** 68dad2a

**5. [Rule 3 - Blocking] Dependencies not installed in worktree**
- **Found during:** Task 1 verification
- **Issue:** Test runner failed because node_modules (including tsx) were not installed in the git worktree
- **Fix:** Ran `pnpm install --frozen-lockfile` + `pnpm --filter @fartola/sportident build`
- **No code change, operational fix**
