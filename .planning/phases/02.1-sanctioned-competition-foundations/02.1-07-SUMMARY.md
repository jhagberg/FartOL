## Decisions Made

- **Auto-push via markDirty wrap:** ProjectionStore has no event emitter; wrapping the method is minimal-overhead and keeps the store interface unchanged. The queue's getConfig callback is called on every push attempt so credential updates via PATCH take effect without restarting the server.
- **15s debounce default:** Coalesces card reads during a busy finish funnel (Gemini HIGH review concern). The operator can trigger immediate pushes via the POST endpoint. Interval is configurable in createPushQueue for tests.
- **In-memory queue accepted for Phase 2.1:** Recovery is natural (next card_read re-triggers), liveresultat is idempotent (re-pushing same snapshot is safe), and operator has manual trigger. Phase 2.2 candidate: SQLite-backed queue.
- **Status endpoint:** Addresses GPT+Gemini HIGH review concern — operators can now see lastPushAt/lastSuccessAt/lastError without grepping logs.
