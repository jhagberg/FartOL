<interfaces>
Walk-up modal route — /competition/[id]/walkup?card=<n>:
- The card query param pre-fills the Bricka field.
- Mounts the WalkupModal overlay over the readout view (if user navigated from readout, history persists; back navigation returns there).

Form state (Svelte 5 runes): name, club, classId, cardNumber (pre-filled from ?card), consent (default true). On Save: POST api.createCompetitor with consent: true literal. On 409 'card_taken': banner offering replace-card flow (plan 10 endpoint).

Subsequent unknown cards queue (UI-SPEC §"Walk-up modal"): readout's pendingUnknownCards grows in background; walk-up modal does NOT auto-switch. Operator clicks history rows to open walkup for each.

Results view — /competition/[id]/results:
- On mount: api.getResults → state.classes; subscribe resultsChannel(competitionId).
- results_full envelope → replace entirely + set updatedAt.
- results_update envelope → patch the matching class; flag new rows with .new class for 4s; set updatedAt.
- Active class tab: first one with rows > 0 default; operator-selectable.
- Fullscreen toggle: F key + button → scales typography (UI-SPEC §"Fullscreen mode": plc 22px, name 18px, time 22px).

Club autocomplete: `<input type="text" list="clubs-list">` + `<datalist>` from api.getClubs with 200ms debounce. Free-text allowed (UI-SPEC §"Walk-up modal").
