**Install-smoke test approach (C-H4 LOCKED — true global-install with --prefix + -g):**

- apps/edge/scripts/install-smoke.sh:
  1. Creates a tmpdir under /tmp/fartola-install-XXXXX.
  2. Runs `npm install --prefix "$TMPDIR" -g --silent "$TARBALL"`. The combination of `--prefix` AND `-g` places the binary at `$TMPDIR/lib/node_modules/.bin/fartola` (the global-prefix layout).   3. Verifies BIN exists at `$TMPDIR/lib/node_modules/.bin/fartola` AND is executable BEFORE invocation; prints the resolved path for transparency; on failure, lists both candidate directories for debug.
  4. Runs the binary with --port 30001 --db-path --no-bridge &amp;.
  5. Sleeps 2s, curls http://127.0.0.1:30001/api/health → expect 200.
  6. Curls http://127.0.0.1:30001/ → expect 200 with HTML containing "fartOLa".
  7. Kills the fartola process, cleans up.

The script is bash + a node:test wrapper. tests/install/install-smoke.test.ts calls the script via child_process AND independently asserts the BIN-path layout invariant (the C-H4 regression gate — if a future npm version changes the global-prefix layout, this test fails before the full smoke runs).
