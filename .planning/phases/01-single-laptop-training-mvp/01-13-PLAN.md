```

  competition: null as CompetitionDTO | null,

  manualDnfDialog: null as { competitorId: string; defaultReason: string } | null,

```

WS subscription on mount: wsClient.subscribe(`readout:${competitionId}`). Handle envelope types:
- card_read: push history (cap 12); set currentRead; trigger flashIn class for 1.6s.
- card_inserted, card_removed: ignored at readout-view level (decorative — plan 13 doesn't show them).
- card_bound: update competitors map; re-derive history rows' competitor binding.
- manual_dnf / un_dnf: re-fetch GET /api/competitions/:id/readout (cheaper than maintaining local projection mirror) OR rely on results: channel updates if also subscribed.
- connection_changed: update stationStatus.
- frame_error: append a transient toast (`Frame error: <reason>`) with 5s timeout.

REST: on mount, GET /api/competitions/:id (for competition + classes) + GET /api/competitions/:id/readout (history + pendingUnknownCards) + GET /api/competitions/:id/competitors (competitors map).

Density (consumed from tweaks store):
- tweaks.density === 'low': LatestReadCard uses 64px runner number, 34px name, 52px time. Show PunchGrid.
- tweaks.density === 'med': default sizes (44px number, 26-28px name, 36px time). Show PunchGrid.
- tweaks.density === 'high': Show SplitsTable instead of PunchGrid. Number stays 44px.

Skogis generator (deterministic):
- Inputs: cardNumber: number, name: string, club: string | null, classId: string.
- Output: SVG element data — viewBox + path/circle/polygon/text elements + an array of stats { fart, stig, kart, tur } each 0..100.
- Hash: `seed = hash(cardNumber + name + club + classId)` (any deterministic 32-bit hash from a stable input string).
- Seeded RNG: simple LCG (mulberry32 or splitmix32). Use the SAME RNG sequence regardless of platform.
- Render rules (mono-printable, chat2.md):
  - Black outlines (1.5px stroke).
  - Paper-colored fills (white / very light --bg-sunken).
  - The "gold crown" is hatched (parallel diagonal lines) when it's the top result; absent otherwise.
  - 1 of N body shapes (squirrel-like / fox-like / owl-like) chosen by seed.
  - 4 stats appended in a grid below the figure (FART = speed, STIG = climb, KART = map, TUR = luck — playful kids language).
- The procedure must be unit-testable: same inputs → same SVG path strings. apps/web/src/lib/skogis/skogis.test.ts asserts determinism.

ReceiptPaper emulates 80mm thermal column: 300px wide, monospace where appropriate, dashed separators (1.5px), receipt typography from tokens.css. UI-SPEC §"Receipt-specific typography".

The 6 templates each take props: `competitor: CompetitorView`, `competition: CompetitionDTO`, `class: ClassDTO`, `course: CourseDTO`, `placeContext?: { place, behind_leader_ms, leaderName? }`. Each template renders its receipt body. Sourced verbatim from `01-SKETCHES/.../screens-readout.jsx` and `print-app.jsx`.
