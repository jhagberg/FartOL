Readout state (Svelte 5 runes):
```
let state = $state({
  competition: null as CompetitionDTO | null,
  classes: [] as ClassDTO[],
  competitors: new Map<string, CompetitorDTO>(),
  currentRead: null as HistoryEntry | null,
  history: [] as HistoryEntry[],
  pendingUnknownCards: [] as number[],
  selectedTemplate: 'classic' as 'classic' | 'standing' | 'detailed' | 'top4' | 'minimal' | 'kids',
  stationStatus: 'opening' as 'opening' | 'open' | 'error',
  manualDnfDialog: null as { competitorId: string; defaultReason: string } | null,
});
```

WS handlers: card_read pushes history (cap 12) + sets currentRead + triggers flashIn for 1.6s. card_inserted / card_removed ignored at readout-view level. card_bound updates competitors map. manual_dnf / un_dnf re-fetch /api/competitions/:id/readout. connection_changed updates stationStatus. frame_error appends a 5s transient toast.

REST on mount: GET /api/competitions/:id (competition + classes) + GET /api/competitions/:id/readout (history + pendingUnknownCards) + GET /api/competitions/:id/competitors.

Density: tweaks.density === 'low' or 'med' → PunchGrid; 'high' → SplitsTable.

Skogis generator (deterministic): inputs cardNumber+name+club+classId; output SVG descriptor + stats {fart, stig, kart, tur} + species + level. Hash + mulberry32 RNG. Mono-printable invariant.

ReceiptPaper: 300px-wide 80mm thermal emulation.

The 6 templates each take props: competitor, competition, classObj, course, placeContext.

**Walk-up trigger (C-M3 LOCKED — readout-query-param variant):**

When a card_read with an unknown card_number arrives:
1. ReadoutView's WS handler appends the card_number to pendingUnknownCards.
2. If pendingUnknownCards.length === 1 AND the card_number IS the just-arrived one, AFTER 600ms (UI-SPEC) the view calls `goto(\`/competition/${id}/readout?walkup=${cardNumber}\`)` — same route, query param added.
3. ReadoutView reads `$page.url.searchParams.get('walkup')` as a reactive `$derived`. When it's non-null, plan 14 mounts `<WalkupModal />` as an OVERLAY on this view (the readout layout is still visible underneath the scrim — UI-SPEC §"Walk-up modal"). NO new SvelteKit route is created.
4. Avbryt / save in the modal calls `goto(\`/competition/${id}/readout\`)` (no query param) — naturally returns to readout proper, history preserved.
5. If multiple unknown cards queue up while the modal is open, the operator clicks history rows to open walk-up for each (UI-SPEC §"Walk-up modal" subsequent-cards behavior).

This route-shape decision is locked to plan 14 in lockstep — plan 14 removes the `/walkup` route file from its files_modified list.
