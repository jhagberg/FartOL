Append-only enforcement (REQ-EVT-002) — C-H1 LOCKED: triggers live in `apps/edge/drizzle/0001_append_only_triggers.sql`, a hand-authored migration file checked into the repo. drizzle-kit does NOT generate triggers from schema-as-TS, and the planner's earlier scheme of appending the triggers to 0000_initial.sql after generation is fragile — any future `db:generate` regenerates 0000 and erases the appended triggers. The 0001 file is never regenerated by drizzle-kit (numbered migrations are immutable once committed), so it survives every regeneration of 0000.

The 0001 file body:
```
-- 0001_append_only_triggers.sql
-- Authored for fartol. Hand-authored, not generated by drizzle-kit.
-- Enforces REQ-EVT-002 (events table append-only). See ADR-0003.
-- Co-versioned with 0000_initial.sql; drizzle-kit never regenerates this file.
CREATE TRIGGER IF NOT EXISTS events_no_update BEFORE UPDATE ON events BEGIN SELECT RAISE(ABORT, 'events table is append-only'); END;
CREATE TRIGGER IF NOT EXISTS events_no_delete BEFORE DELETE ON events BEGIN SELECT RAISE(ABORT, 'events table is append-only'); END;
```

The drizzle-kit `meta/_journal.json` must list BOTH 0000_initial and 0001_append_only_triggers so the migrator applies both on cold start. drizzle-kit's `generate` will only write the 0000 entry automatically; the executor manually appends an entry for 0001 in `_journal.json` with a fixed `when` timestamp and a fresh `tag` value. Document the exact JSON shape in the summary; the runtime migrator (drizzle-orm/better-sqlite3/migrator) walks `_journal.json` numerically.

The retention scrub (REQ-PRIV-002, plan 17) UPDATEs *competitors* (not events) — the events table stays sealed. The competitor's name + club columns are nulled / replaced with the anonymized placeholder. The card_number stays (it's a hardware ID, not PII per research.md §6).

    import crypto from 'node:crypto';

    export function ensureNodeId(handle: DbHandle): string {
      const existing = handle.db.select().from(config).where(eq(config.key, 'node_id')).get();
