## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02P3-01 | Spoofing | malicious LAN device polls /mip and harvests competitor PII (name, club) | accept | D-MIP-1 explicit no-auth posture for 4-klubbs; Phase 2.1 adds pwd header check |
| T-02P3-02 | Tampering | malicious LAN device floods /mip with high-frequency requests | accept | Pitfall 9 — defer rate limiting; if it manifests at bench, add @fastify/rate-limit scoped to /mip only |
| T-02P3-03 | Info Disclosure | hired_contact details leak via MIP `<entry>` | mitigate | MIP entry ONLY includes the boolean hired flag; contact_* columns NEVER serialized to MIP per D-HB-1 (it's our extension; MeOS doesn't model it) |
| T-02P3-04 | Tampering | malformed lastid (decimal, negative, string) causes infinite loop or panic | mitigate | Zod schema rejects with 400; Landmine "input.php lastid coercion" — we're stricter than the PHP reference |
| T-02P3-05 | Info Disclosure | pino logs leak full competitor rows on each /mip poll | mitigate | Log only `{ lastid, entry_count }` not full payload; Fastify default log shape doesn't dump body |
| T-02P3-06 | Tampering | XML special chars in competitor name break MIP wire format | mitigate | XMLBuilder auto-escapes `<>&"'` per fast-xml-parser docs; verified by test 3 using a synthetic name containing `&` |
| T-02P3-SC | Tampering | no new dependencies added | mitigate | Reuses Phase 1's fast-xml-parser + xmllint-wasm; no install gate needed |
