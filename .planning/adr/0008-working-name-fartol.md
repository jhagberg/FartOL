# Working name "fartol"

## Context and Problem Statement

The project needs a short identifier for the repo, CLI binary, and
package names before public release. Pick a final name now, or use a
working name?

## Considered Options

- Pick a final name now and commit to it
- Use a working name; rename at first public release

## Decision Outcome

Chosen option: **use the working name `fartol`**, because repo
renaming is a one-button operation on GitHub, finalizing a public name
needs brand/legal review that shouldn't block development, and
bikeshedding the name now is pure delay. Rename before public release.
